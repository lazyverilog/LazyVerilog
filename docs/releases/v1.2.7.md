# LazyVerilog v1.2.7 Release Notes

Safer code-action applies and stricter configuration validation.

## Fixes

- Fixed stale indentation inside `else if (...) begin ... end` blocks after formatting.
- Fixed AutoFF apply sometimes inserting stale text, neighboring text, or nothing even though the preview window showed the correct flip-flop assignments.
- Fixed AutoWire apply using stale coordinates after formatting.

## Configuration

- `lazyverilog.toml` values are now sanity-checked on load.
- Documentation and the example `lazyverilog.toml` are now aligned with the supported source options.
