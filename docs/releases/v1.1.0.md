# LazyVerilog v1.1.0 Release Notes

Navigation and responsiveness release.

## Changes

LazyVerilog now keeps a cached structural index for each open document snapshot and reuses it across references, RTL tree, inlay-hint, lint, workspace-symbol, and code-generation paths. Project-file caches also use direct URI lookup instead of repeated linear scans, and formatter save/edit paths avoid several avoidable rebuilds and reallocations.

## Fixes

- Find References now works on SystemVerilog macros, including both `` `define NAME `` declarations and `` `NAME `` invocation sites.
- Same-line incremental edits no longer delete text after the cursor. This fixes save-time corruption where typing a macro dimension such as `` logic [`PARAMSVH_PARAM-1:0] include_param; `` could drop the closing `]`.
- WorkspaceEdit JSON generation now escapes URIs and replacement text consistently.
- Inlay hints are refreshed after background project indexing publishes module definitions, so hints for filelist-defined instances appear after startup without requiring a code edit.
- Formatter control-comment regexes are compiled once per formatting run instead of once per token.
