# LazyVerilog v1.2.5 Release Notes

## Hover Improvements

- Hovering over a function or task defined in a closed project file now shows its signature correctly.
- Hovering over a module instance in a closed project file now shows the port list.
- Hovering over symbols defined in `\`include` header files now resolves and displays correctly.

## Find References Improvements

- Find References now works for symbols defined in `\`include` header files — results are no longer missing when the symbol lives in an included file rather than the file where it is used.
- Class member calls inside included files are now indexed and appear in Find References results.
- Subroutine calls that cross `\`include` boundaries are now matched correctly — calling a task or function defined in an included file no longer produces missing or incomplete results.

## AutoWire Fixes

- Fixed a crash that occurred on certain modules (null header dereference in module range finder).
- Fixed AutoWire picking up signals from `\`include` header files and incorrectly treating them as belonging to the current module.
- AutoWire now inserts only the new wire/port lines without reformatting the rest of the file.

## AutoFF Fix

- Inserted flip-flop assignments are now properly formatted before being written into the document.

## Formatter Fix

- Variable declaration alignment groups are now preserved across comment-only lines. Previously, a lone comment between declarations broke the alignment group, causing misaligned output.

## File Watching

- File changes on disk are now detected immediately via inotify interrupt instead of waiting for a polling cycle. Deleted or externally modified files update faster in the editor.
