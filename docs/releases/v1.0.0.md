# LazyVerilog v1.0.0 Release Notes

LazyVerilog v1.0.0 is the first public release of the C++ LazyVerilog language server and Neovim integration.

This release focuses on practical SystemVerilog RTL editing: fast formatting, configurable lint diagnostics, project-aware navigation, and code-generation helpers for common hardware-development workflows.

## Highlights

- C++ SystemVerilog Language Server Protocol server.
- Neovim plugin with clean setup through `require("lazyverilog").setup()`.
- Configurable formatter with safe-mode checks to prevent non-whitespace content changes.
- RTL/style lint diagnostics controlled by `lazyverilog.toml`.
- Project-aware editor features: hover, completion, signature help, definition, references, rename, workspace symbols, and inlay hints.
- RTL productivity commands including auto-instantiation, auto-argument, auto-function, auto-wire, auto-FF, connect helpers, and interface helpers.
- RTL tree and reverse RTL tree views.
- Formatter documentation and macro-formatting policy documentation.
- Large open-source RTL corpora added as test submodules for formatter validation.

## Formatter

The v1.0.0 formatter is token-based and pass-oriented. It supports configurable policy for:

- indentation and blank lines,
- assignment spacing and alignment,
- declaration and port alignment,
- instance port alignment,
- function/task call wrapping,
- function/task declaration layout,
- module headers and module parameter layout,
- enum and modport alignment,
- macro formatting categories,
- format-off / format-on regions.

The formatter also includes `safe_mode`, which verifies that formatting did not change non-whitespace content.

## Configuration

LazyVerilog uses a project-local `lazyverilog.toml`. The server walks upward from opened files to find the nearest config file.

Configuration covers:

- design file list and preprocessor defines,
- background semantic compilation,
- lint rules,
- formatter behavior,
- RTL tree display,
- inlay hints,
- auto-function / auto-arg / auto-wire behavior.

## Neovim Installation

Using `lazy.nvim`:

```lua
{
  "lazyverilog/LazyVerilog",
  submodules = false,
  ft = { "systemverilog", "verilog" },
  config = function()
    require("lazyverilog").setup()
  end,
}
```

The plugin resolves the LSP binary in this order:

1. explicit `setup({ cmd = ... })`,
2. `lazyverilog-lsp` on `$PATH`,
3. plugin-managed binary under Neovim's data directory,
4. release-binary auto-download.

## Build From Source

```bash
cmake -B build
cmake --build build -j$(nproc)
```

Run tests:

```bash
ctest --test-dir build
```

## Notes for Release Packaging

Release assets should match the Neovim plugin downloader naming scheme:

```text
lazyverilog-lsp-v1.0.0-linux-x64
lazyverilog-lsp-v1.0.0-linux-arm64
lazyverilog-lsp-v1.0.0-darwin-x64
lazyverilog-lsp-v1.0.0-darwin-arm64
```

## Known Notes

- The repository contains large RTL test corpora as submodules. End users installing through `lazy.nvim` should set `submodules = false`.
- If testing a local server build, pass `cmd` explicitly or use:

```bash
LAZYVERILOG_LSP=build/lazyverilog-lsp tools/run_clean_nvim.sh demo/memory_top.sv
```
