# LazyVerilog v1.2.0 Release Notes

Project-index, Connect, formatter, and LintAll reliability release.

## Changes

LazyVerilog now reuses more indexed project structure across request paths instead of rebuilding or merging whole-project state on demand. Open buffers publish index shards from their live AST snapshots, closed filelist files stay index-backed, and references, goto definition, hover, diagnostics, RTL tree, inlay hints, and code-generation paths share more cached syntax facts. This improves startup and request responsiveness while preserving the design rule that current/open files use live AST data and closed project files do not retain full ASTs.

The Neovim integration adds a client-side `:LintAll` command with progress reporting, improves RTL tree navigation, and avoids stray lint split buffers. Connect routing and preview behavior have been refined for cross-hierarchy edits, and formatter safety is now always enforced for both direct formatting and save-time AutoArg edits.

## Fixes

- Connect now routes cross-hierarchy signals through explicit boundary ports and merges multiple instance-port edits into stable, non-overlapping WorkspaceEdits.
- `:LintAll` reports progress, keeps background compilation priority in sync, and logs diagnostic URI failures instead of silently dropping them.
- Find References uses symbol- and source-location-indexed facts, fixing stale snapshot reuse, package-scope misses, module-local subroutine scoping, and closed-file false positives.
- Diagnostics clear when documents close and avoid merging the project index on hot diagnostic paths.
- Inlay hints for ports are less noisy, show port direction in their icons, and handle package scope more accurately.
- Formatter safety no longer depends on a user-facing `safe_mode` option; formatted output must preserve non-whitespace content and the stripped token stream before edits are returned.
- Formatter fixes include conditional preprocessor indentation, stale formatting around module-local tasks/functions, fork/join indentation, enum adaptive alignment docs, function-declaration parenthesis spacing, and alignment regressions.
- Folding ranges handle continuous port declarations, avoid overlaps with module parameter and port-list ranges, and do not leave stale ranges around conditional preprocessor regions.
- AutoArg no longer applies duplicate save-time formatting, and stale AutoInst lint checks use current-file AST facts.
- Config validation preserves structured errors, repeated naming/formatter regexes are cached, and large whole-document edits build JSON with fewer reallocations.
