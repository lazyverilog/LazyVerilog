# LazyVerilog v1.0.6 Release Notes

Folding-range refinement release.

## Changes

LazyVerilog now provides more predictable SystemVerilog folding ranges for modules, functions/tasks, classes, covergroups, generate blocks, case blocks, comments, imports, and preprocessor regions. Preprocessor folds are based on slang directive syntax/trivia instead of raw source string matching, and inactive branches recover local `begin`/`end` folds from disabled tokens where possible.

## Fixes

- Folding on `ifdef`, `elsif`, and `else` now folds the intended preprocessor branch instead of falling through to the whole module.
- `ifdef` branches fold up to the next branch directive; final branches fold through the closing `endif`.
- Trailing comments no longer fold RTL code lines.
- Duplicate folding ranges are removed before returning LSP results.
