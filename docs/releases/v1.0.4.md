# LazyVerilog v1.0.4 Release Notes

Bug-fix release.

## Changes

- **Connect / Interface** — implemented the previously advertised Connect and Interface execute commands in the C++ server, including Neovim preview/apply flows for wiring, interface inspection, connect, and disconnect.
- **AutoArg** — generated non-ANSI module headers are now passed through the main formatter before being returned/applied, so AutoArg output follows the same formatting pipeline as the other auto-code features.
- **AutoFF** — `register_pattern` moved from `[lint.naming]` to a dedicated `[autoff]` section. Update your `lazyverilog.toml` if you set this option explicitly.

## Fixes

- **Connect / Interface** — generated bridge signal declarations now use the selected output port as the declaration source, generate `logic` for net-style output ports such as `output wire [5:0]`, and preserve user-defined datatypes plus macro/parameter dimensions.
- **Interface** — fixed table alignment when two displayed instances have no shared connections.
- **AutoFF** — fixed preview/apply handling so preview commands return confirmation data for the Neovim floating window, while apply commands return workspace edits.
- **AutoFF** — fixed `always_ff @(...) begin ... end` detection by unwrapping timing-control syntax; clock/reset signal names remain irrelevant.
- **AutoFF** — ambiguous two-signal declarations are now skipped instead of guessing from declaration order. A declaration must contain exactly one signal matching `[autoff].register_pattern` and exactly one non-matching source signal.
- **Hover** — fixed empty hover responses to serialize as a top-level JSON `null` result instead of `{"contents": null}`. This keeps the response shape aligned with the LSP `Hover | null` contract and avoids a Neovim 0.11 hover crash when hovering locations without symbol information.
