# LazyVerilog v1.1.1 Release Notes

Folding-range usability release.

## Changes

LazyVerilog now provides more useful folding ranges for parameterized module headers. Parameter lists and port lists can be folded separately, while the enclosing module fold still covers the full module from `module` through `endmodule`. Consecutive declaration sections are also broader and more consistent, including user-defined type declarations and non-ANSI port declarations.

## Fixes

- Folding on a parameterized module header no longer collapses the whole module when the intended target is the `#(...)` parameter list or following port list.
- Parameter-list and port-list folds no longer hide behind a redundant whole-header fold.
- Folding from inside a module body uses the whole module range, including the header.
- Consecutive non-ANSI port declarations now fold like other declaration runs.
- Declaration folding now includes user-defined type declarations instead of only keyword-led declarations.
- Folding ranges from included files are no longer reported in the current document.
