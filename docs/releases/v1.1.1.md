# LazyVerilog v1.1.1 Release Notes

Connect routing and formatter reliability release.

## Changes

LazyVerilog's Connect command now routes signals through explicit hierarchy boundary ports instead of silently reusing the bridge-wire name at every level. The Neovim flow asks for source-side and destination-side boundary ports, previews the full multi-file route, and applies merged instance edits so empty instances such as `u_inv ();` become stable named-port connections in one edit. Closed filelist sources can also participate in Connect edits without retaining full ASTs: open buffers still use live AST snapshots, while project files use `SyntaxIndex` plus on-demand file text for WorkspaceEdits.

The formatter and request paths also received several safety and performance fixes, including cheaper formatter lexing metadata, cached AutoFF regex compilation, safer large-document end-position handling, and more accurate formatter metadata for covergroups and argument-list context.

## Fixes

- Cross-hierarchy Connect now updates leaf instances, intermediate boundary ports, and the LCA bridge wire consistently across multiple files.
- Connect no longer depends on dynamic signal-name regexes for declaration/type checks, avoiding regex injection and multiline-declaration misses.
- Connect merges multiple requested port changes for the same instance into one non-overlapping edit.
- Connect can edit closed filelist files by reading source text on demand while keeping closed-file structure index-based.
- Formatter debug logging now honors `format.log_path` from `lazyverilog.toml`.
- Large-document whole-file edit ranges no longer rely on unchecked `int` casts for end positions.
- Formatter token collection avoids avoidable lower-case text allocations for normal non-directive tokens.
- AutoFF caches repeated register-pattern regex compilation.
